Current Project: Programming

Initial Request

In an effort to provide meals to our users, we would like to provide the ability to offer peanut butter and jelly sandwiches. The users should be able to select a sandwich, and pick it up at the kitchen in the cafeteria when it is ready.

Design Documentation

Purpose

This functionality will allow users to select a meal and automatically send it to the kitchen where it will be prepared and waiting. Users will be able to select the types of foods they would like, while specified admin users will be able to set up the options that are available for a given day or a given meal. The current set up will allow users to choose a single meal type - a Peanut Butter and Jelly sandwich - with several options, but this system is being designed in order to allow for additional offerings in the future.

Usage

General Information

Users of the system will be able to order a meal. Currently there is one meal option available - Peanut Butter and Jelly (PBJ) sandwich. There are various customizations available for the sandwich, which can be toggled as needed by the pre-set system adminstrators. The system is designed so that in the future, additional menu items and customizations can be added.

Adminstration

A system adminstrator will be able to set up customizations for a given meal. The list of any available items will be available on the new Admin screen, and each item can be selected. A new section will appear that will include customizations that have been set up for that particular food item, and each customization can be toggled on or off depending on availability of the selection. For some customizations, individual items can be toggled or changed.

For the peanut butter and jelly, there are several customizations:

  • Bread: Two types of bread will be available to start with - Whole Wheat and White. Adminstrators will be able to add additional types of bread or remove existing ones, as well as toggle the availability of all types on or off. Note that at least one type of bread must be available and toggled on or the data cannot be saved. The bread can also be further customized:
    • Toasted: Yes or No option, determines whether the bread will be toasted.
    • All customizations are able to be toggled on or off individually, as well as having new options added or existing options removed.
  • Peanut Butter: Three types of peanut butter will be available to start with - Chunky Peanut, Smooth Peanut, or Almond. Adminstrators will be able to add additional types of peanut butter or remove existing ones, as well as toggle the availability of each individual on or off. Additional customization options will be available, which can individually be toggled on or off:
    • Amount: Light, Regular, or Extra options, determines how much peanut butter will be on the sandwich.
    • All customizations are able to be toggled on or off individually, as well as having new options added or existing options removed.
  • Jelly: Two types of jelly will be available to start with - Jelly or Jam. Adminstrators will be able to add additional types of jelly or remove existing ones, as well as toggle the availability of each individual on or off. Additional customization options will be available, which can individually be toggled on or off:
    • Amount: Light, Regular, or Extra options, determines how much jelly will be on the sandwich.
    • Flavor: Strawberry, Grape, Mixed Fruit, determines the flavor of jelly or jam.
    • All customizations are able to be toggled on or off individually, as well as having new options added or existing options removed.
  • Drink: Three types of drinks will be available to start with - Juice, Milk, or Soda. Adminstrators will be able to add additional types of drinks or remove existing ones, as well as toggle the availability of each individual on or off. Additional customization options will be available, which can individually be toggled on or off:
    • Juice: If Juice is selected, then the options available will start with Apple, Orange, Cranberry.
    • Milk: If Milk is selected, then the options available will start with Chocolate, Vanilla, Strawberry, Regular, or Almond.
    • Soda: If Soda is selected, then the options available will start with Coke, Diet Coke, Coke Zero, Cherry Coke, Cherry Coke Zero, Sprite, Orange Soda, Root Beer.
    • Size: Small, Medium, Large options, determines how much drink will be provided.
    • All customizations are able to be toggled on or off individually, as well as having new options added or existing options removed.

User Functionality

A user will be able to access the screen and select options for a peanut butter and jelly sandwich. Several things are able to be customized, with options based on what the system adminstrators have selected or toggled.

  • Bread: (Required) User will need to choose an option for bread. The following customizations are available:
    • Bread Type will determine what type of bread is used. INITIAL: WHOLE WHEAT, WHITE.
    • Toasted will determine whether the bread should be toasted (Y) or not (N). OPTIONS: Y, N.
    • NOTE: If the bread types have not been set up by an adminstrator, or if none are remaining or toggled on, then the PBJ option will be unavailable.
  • Peanut Butter: User will need to choose an option for peanut butter. The following customizations are available:
    • Peanut Butter Type will determine what type of peanut butter is used. INITIAL: CHUNKY, SMOOTH, ALMOND.
    • Amount will determine how much peanut butter should be used. Light will be 50% of the regular amount, and extra will be 150%. OPTIONS: LIGHT, REGULAR, EXTRA.
    • NOTE: Either Peanut Butter or Jelly must be selected. A user is allowed to choose only one.
  • Jelly: User will need to choose an option for Jelly. The following customizations are available:
    • Jelly Type will determine what type of jelly is used. INITIAL: JELLY, JAM.
    • Flavor will determine what flavor of jelly should be used. OPTIONS: STRAWBERRY, GRAPE, MIXED FRUIT.
    • Amount will determine how much jelly should be used. Light will be 50% of the regular amount, and extra will be 150%. OPTIONS: LIGHT, REGULAR, EXTRA.
    • NOTE: Either Peanut Butter or Jelly must be selected. A user is allowed to choose only one.
  • Drink: (Optional) User will be able to choose an option for a drink. User can choose between three drink types initially, with additional options becoming available once an initial selection is made. The following customizations are available:
    • JUICE: If the initial option of JUICE is chosen, then the user will be able to select a type of juice. INITIAL: APPLE, ORANGE, CRANBERRY.
    • MILK: If the inital option of MILK is chosen, then the user will be able to select a type of milk. INITIAL: REGULAR, VANILLA, CHOCOLATE, STRAWBERRY, ALMOND.
    • SODA: If the inital option of SODA is chosen, then the user will be able to select a type of soda. INITIAL: COKE, CHERRY COKE, DIET COKE, COKE ZERO, CHERRY COKE ZERO, SPRITE, ORANGE SODA, ROOT BEER.
    • Size will determine how much of the drink should be provided. Small will be 12oz, Medium will be 16oz, and Large will be 20oz. OPTIONS: SMALL, MEDIUM, LARGE.

Technical Functionality

Connection Information

The system will send the information into the kitchen via the higgledy-piggledy API server. A request will be sent, and when the server receives the response and successfully sends it to the server in the kitchen, a response will be generated and sent back with one of a list of response codes. Once the kitchen receives the request, that server will translate the request and print it out so it can be created by others and served up waiting.

Request & Response

The request will be formatted as a JSON string with the information input from the user as the data points.

{
 "BreadType": "WholeWheat",
 "Toasted": "Y",
 "PeanutButterType": "Chunky",
 "PeanutButterAmount": "Regular",
 "JellyType": "Jelly",
 "JellyFlavor": "Grape",
 "JellyAmount": "Regular",
 "Drink": "(Soda, Coke Zero)",
 "DrinkSize": "Medium"
}

The response will be formatted as a JSON string, and will send back a request based on a key of response codes

{
 "ResponseCd": "100",
 "Error": "None"
}

Possible Response Codes:

Response Code Meaning
100 Success
101 System Unavailable
102 Issue With Request
